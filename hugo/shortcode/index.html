Q: 以下程式在 Markdown 中 用途為何?



...

A:

在 Hugo 中， ` 和 ` 是用來插入原始 HTML 內容的短代碼（shortcode）。它的作用是允許在 Markdown 文件中嵌入原始 HTML，而不會被 Hugo 進一步處理或修改。

功能與用途

  1. 嵌入原始 HTML

    • 在 Hugo 渲染 Markdown 文件時，這些標籤之間的 HTML 內容會被直接輸出到最終的 HTML 中，而不會被 Hugo 調整或解釋。
    • 適合用於插入自定義 HTML 元素（如內嵌樣式、腳本或其他特殊標籤）。

  2. 保護 HTML

    • 如果不使用 rawhtml，Hugo 可能會對 HTML 進行轉義或過濾，導致原始內容無法正常顯示。使用 rawhtml 可以避免這種情況。

使用範例

嵌入一段自定義的 HTML

假設你想在文章中插入一段自定義的表格，可以這樣使用：





  

    Header 1
    Header 2
  

  

    Data 1
    Data 2

在最終生成的 HTML 中，這段表格會被直接輸出並正確顯示。

插入 JavaScript

如果需要插入 JavaScript 片段：

這段腳本會直接嵌入到最終的 HTML 頁面中。

注意事項

  1. 安全性：

    • 如果你啟用了 Hugo 的 safeHTML 選項，只有受信任的原始 HTML 內容會被保留，其他可能被過濾。
    • 在公開網站中嵌入外部 HTML 或腳本時，需確保其來源是安全的，避免潛在的 XSS 攻擊。

  2. 內容處理限制：

    • 在 rawhtml 標籤內的內容不會經過 Markdown 處理。如果需要混合使用 Markdown 和 HTML，應謹慎設計。

  3. 使用條件：

    • 某些情況下，直接寫 HTML 而不使用 rawhtml 也能正常渲染，但如果發現 Hugo 對 HTML 進行了意外的轉義或修改，可以嘗試使用 rawhtml 短代碼。

總結來說， ` ` 是一種方便的工具，用於確保特定的 HTML 內容能夠不受干擾地嵌入到 Hugo 生成的頁面中。

Q: 這跟我在 shortcodes/rawhtml.html 裡面的 {{ .Inner }} 有何關聯?

A:

{{ .Inner }} 是 Hugo 中 shortcodes 的內建變數之一，它與 ` 和 ` 之間有直接的關聯。讓我們具體解釋它們的關係及其運作原理。

shortcodes/rawhtml.html 的作用

當你使用 ... 時，Hugo 會自動查找 shortcodes/rawhtml.html 這個模板文件，並使用該模板來處理標籤內的內容。

{{ .Inner }} 的功能

  1. {{ .Inner }} 表示短代碼標籤內部的內容：
    • {{ .Inner }} 是 Hugo 中 shortcodes 的一個變數，用來代表短代碼的開閉標籤之間的所有內容。
    • 在 shortcodes/rawhtml.html 中，Hugo 會將 ` 和 ` 之間的文本（可能是 HTML）傳遞給 {{ .Inner }}。

工作原理

當你在 Markdown 文件中這樣寫：





  
Example

Hugo 會執行以下步驟：

  1. 匹配模板：找到 shortcodes/rawhtml.html 作為對應的模板。
  2. 替換 {{ .Inner }}：
    • {{ .Inner }} 被替換為 ` 和 ` 之間的內容： 
      <table>
  <tr><td>Example</td></tr>
</table>
  3. 生成輸出：將模板中包含的內容嵌入到最終的 HTML 中。

shortcodes/rawhtml.html 文件範例

如果你的 shortcodes/rawhtml.html 文件內容如下：

{{ .Inner }}

Markdown 文件的輸出結果將是：

<table>
  <tr><td>Example</td></tr>
</table>

進一步的自定義

你可以在 shortcodes/rawhtml.html 文件中對 {{ .Inner }} 的內容進行包裹或處理。

範例：包裹一層 <div>

假設你希望自動在所有 rawhtml 的內容外包裹一層 <div>，可以這樣修改模板：

<div class="raw-html-content">
  {{ .Inner }}
</div>

輸出結果將是：

<div class="raw-html-content">
  <table>
    <tr><td>Example</td></tr>
  </table>
</div>

如何應用這些關係

  • 如果你不修改 shortcodes/rawhtml.html，Hugo 只會將 {{ .Inner }} 的內容直接輸出，這就是默認的 rawhtml 行為。
  • 如果你需要對 {{ .Inner }} 的內容進行包裹、過濾或添加其他 HTML 元素，可以通過修改 shortcodes/rawhtml.html 實現。

結論

  • {{ .Inner }} 是 rawhtml 短代碼中的核心部分，它直接決定了 ... 標籤內部的內容如何被渲染。
  • 你可以使用 shortcodes/rawhtml.html 文件來自定義這些內容的處理方式，例如包裹、格式化或增加額外樣式。

Q: 所以 rawhtml 是一個自定義的 shortcode? 我也可以換個名稱嗎?

A:

是的，rawhtml 是一個自定義的 shortcode。你完全可以修改名稱或新增一個自定義的 shortcode，來實現與 rawhtml 類似或完全不同的功能。

如何自定義 Shortcode 名稱

1. 修改 Shortcode 名稱

如果你想把 rawhtml 改成其他名稱，比如 customhtml，只需以下幾步：

  1. 重命名文件
    將 shortcodes/rawhtml.html 文件重命名為 shortcodes/customhtml.html。

  2. 更新使用方式
    在 Markdown 文件中，使用新的名稱：

    &lcub;&lcub;< customhtml >&rcub;&rcub;
<table>
  <tr><td>Custom HTML Example</td></tr>
</table>
&lcub;&lcub;< /customhtml >&rcub;&rcub;

Hugo 會自動查找並渲染 shortcodes/customhtml.html 文件中的邏輯，作用與原來的 rawhtml 一樣，只是名稱改了。

2. 保留原名稱並新增新功能

如果你想保留 rawhtml 並新增一個不同功能的 shortcode，可以新建另一個文件，比如 shortcodes/myhtml.html。

範例：
假設你的 myhtml.html 模板增加了一些固定的樣式包裹：

<div class="custom-style">
  {{ .Inner }}
</div>

在 Markdown 文件中使用：

&lcub;&lcub;< myhtml >&rcub;&rcub;
<p>This is wrapped in custom styles.</p>
&lcub;&lcub;< /myhtml >&rcub;&rcub;

輸出結果：

<div class="custom-style">
  <p>This is wrapped in custom styles.</p>
</div>

注意事項

  1. Shortcode 名稱是靜態的：

    • 名稱由模板文件名確定，比如 shortcodes/rawhtml.html，因此不能動態命名。

  2. 避免命名衝突：

    • 如果新建的 shortcode 名稱與 Hugo 的內建 shortcode（如 figure、highlight）同名，可能會導致內建功能被覆蓋。

  3. 支援參數傳遞：

    • 自定義 shortcodes 還可以接收參數，進一步實現靈活的功能。

範例：支持一個 class 參數

<div class="{{ .Get "class" }}">
  {{ .Inner }}
</div>

Markdown 使用：

&lcub;&lcub;< myhtml class="highlight" >&rcub;&rcub;
<p>This paragraph has a highlight class.</p>
&lcub;&lcub;< /myhtml >&rcub;&rcub;

輸出結果：

<div class="highlight">
  <p>This paragraph has a highlight class.</p>
</div>

結論

  • 是的，rawhtml 是自定義 shortcode，你可以隨時更改名稱（如 customhtml）。
  • 你也可以創建新的 shortcode，並根據需求設計其邏輯和功能。
  • Hugo 提供了靈活的短代碼機制，方便你管理和定制模板行為。

Q:

  1. Hugo 有哪些內建 shortcode
  2. 所以呼叫 shortcode 的語法是 {{ <要呼叫的 shortcode 名稱> }} 是嗎?

A:

1. Hugo 的內建 Shortcode

Hugo 提供了多種內建的短代碼（shortcode），用於處理常見需求。以下是一些常用的內建 shortcodes：

媒體相關

  • figure：插入圖片或圖表，支持標題與自定義樣式。

  • youtube：嵌入 YouTube 視頻。

  • vimeo：嵌入 Vimeo 視頻。

內容嵌入

  • gist：嵌入 GitHub Gist。

    &lcub;&lcub;< gist user gistID >&rcub;&rcub;

  • tweet：嵌入 Twitter 帖子。

    &lcub;&lcub;< tweet id="1234567890" >&rcub;&rcub;

程式碼顯示

  • highlight：語法高亮顯示程式碼。 
    print("Hello, Hugo!")

數學公式

  • math：插入數學公式（需要額外配置）。 
    &lcub;&lcub;< math >&rcub;&rcub;
E = mc^2
&lcub;&lcub;< /math >&rcub;&rcub;

短鏈接

  • ref 和 relref：生成指向其他頁面的短鏈接。 
    [Link to another page](&lcub;&lcub;< ref "path/to/page.md" >&rcub;&rcub;)

其他

  • param：取頁面的參數值。

    &lcub;&lcub;< param key="exampleParam" >&rcub;&rcub;

  • time：顯示時間或格式化日期。

    &lcub;&lcub;< time >&rcub;&rcub;

完整內建短代碼列表可以參考官方文檔：Hugo Shortcodes Documentation

2. 呼叫 Shortcode 的語法

基本語法

是的，呼叫短代碼的語法如下：

&lcub;&lcub;< shortcodename >&rcub;&rcub;

標籤式 Shortcode

如果該 shortcode 支持內部內容，需使用成對的標籤：

&lcub;&lcub;< shortcodename >&rcub;&rcub;
內容
&lcub;&lcub;< /shortcodename >&rcub;&rcub;

帶參數的 Shortcode

短代碼可以接受參數：

&lcub;&lcub;< shortcodename param1="value1" param2="value2" >&rcub;&rcub;

帶內容與參數的 Shortcode

短代碼也可以同時接收參數與內部內容：

&lcub;&lcub;< shortcodename param1="value1" >&rcub;&rcub;
內容
&lcub;&lcub;< /shortcodename >&rcub;&rcub;

3. 範例對比

呼叫內建 Shortcode

呼叫自定義 Shortcode

&lcub;&lcub;< myshortcode param="value" >&rcub;&rcub;
Custom content inside.
&lcub;&lcub;< /myshortcode >&rcub;&rcub;

結論

  1. Hugo 提供多種內建 shortcodes，如 figure、youtube、highlight 等，用於處理媒體嵌入、程式碼高亮等常見需求。
  2. 呼叫 shortcodes 的基本語法是 &lcub;&lcub;< shortcode >&rcub;&rcub;，如果需要內部內容或參數，可以使用成對標籤並傳遞參數。

Q: 回到剛剛的 rawhtml 這個 shortcode，其內容只有 {{ .Inner }}，相當於內容不改動，那麼跟我把 shortcode 去掉有何差別?

A:

你的觀察是正確的：如果 rawhtml 的 shortcode 模板只包含 {{ .Inner }}，那麼它的作用就是直接輸出 shortcode 標籤內的內容，不進行任何修改。在這種情況下，和不使用 shortcode 將 HTML 直接放在 Markdown 文件中確實效果相同。但它還是有一些隱性功能和好處。

區別與用途

1. 解析與過濾

  • 不使用 shortcode： 如果直接在 Markdown 中插入 HTML，Hugo 會將其解釋為普通 HTML，並可能受到 Markdown 和安全過濾規則的影響。例如：

    <table>
  <tr><td>Example</td></tr>
</table>

    如果安全設置（markup.sanitizeHTML）開啟，某些 HTML 可能會被 Hugo 過濾掉（比如 <script> 或 <iframe>）。

  • 使用 rawhtml shortcode： 短代碼內的 HTML 通常會被 Hugo 認為是“受信任的”，跳過某些安全過濾器。這意味著即使有安全限制，使用 rawhtml 仍能渲染完整的 HTML。

2. 結構化與重用

使用 shortcode 有助於讓內容更有結構，方便重用：

  • 保持內容與行為分離： 如果你在多處需要插入 HTML，通過短代碼統一管理比較方便。例如：

    

  
    
    
    Example

    如果將來需要對 rawhtml 的渲染行為進行修改（比如添加外層 <div>），只需更新 shortcodes/rawhtml.html，而不需要逐個修改 Markdown 文件。

3. 支援內嵌的 Hugo 語法

在 shortcode 中，你可以混合使用 Hugo 的模板語法，這是直接插入 HTML 無法做到的：

  • 範例：條件渲染

    {{ if .Get "table" }}
<table>{{ .Inner }}</table>
{{ else }}
<div>{{ .Inner }}</div>
{{ end }}

    Markdown 文件中：

    

  Example

    輸出：

    <table>
  <tr><td>Example</td></tr>
</table>

4. 提升可讀性與維護性

  • 簡化 Markdown 文件： 在 Markdown 文件中使用 shortcodes，能讓長段 HTML 看起來更簡潔，容易閱讀。例如： 
    

  
    
    
    Example
    比直接嵌入 HTML 更有組織性，尤其是當 HTML 複雜或多行時。

5. 支援參數擴展

即使現在 rawhtml 只是輸出 {{ .Inner }}，未來可以輕鬆擴展，比如支持參數：

  • 範例：

    <div class="{{ .Get "class" }}">
  {{ .Inner }}
</div>

    Markdown 文件：

    

  
    
    
    Example

    渲染結果：

    <div class="table-wrapper">
  <table>
    <tr><td>Example</td></tr>
  </table>
</div>

結論

  1. 功能一致：目前 rawhtml 只包含 {{ .Inner }}，確實和直接寫 HTML 的效果相同。
  2. 擴展性強：使用 shortcode 方便未來擴展行為，比如添加樣式、參數支持或邏輯處理。
  3. 安全與過濾：shortcode 可以避免 HTML 被過濾。
  4. 結構化與可維護性：shortcode 提升內容的可讀性，方便統一管理，適合需要重用的場景。

即使在簡單的場景下，使用 shortcode 仍是一種更靈活的設計方式，特別是當你的網站規模變大時。
